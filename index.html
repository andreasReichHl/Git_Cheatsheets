Mein Cheat Sheet für Git

Git ist eine freie Software zur verteilten Versionverwaltung von Dateien.

Top Eingaben ins Terminal für die Verwendung von git

  • git -v

    • Zeigt die aktuelle Git-Version an, die auf Ihrem System installiert ist.

  • git init

    • Der Befehl wird verwendet, um ein neues Git-Repostory zu initalisieren.

  • git add .

    • Der Befehl wird verwendet, um alle Änderungen in Ihrem Arbeitsverzeichnis zur Staging-Area (auch "Index" genannt) hinzuzufügen, damit sie im nächsten Commit erfasst werden können.

  • git add dateiname

    • Der Befehl wird verwendet, um eine spezifische Datei zur Staging-Area (auch "Index" genannt) hinzuzufügen.Nur diese ausgewählte Datei wird dann in den nächsten Commit aufgenommen.

  • git commit -m "was geändert wurde"

    • Der Befehl wird verwendet, um die Änderungen, die sich in der Staging-Area befinden, in einem neuen Commit zu speichern. Der -m-Schalter ermöglicht es Ihnen, eine Commit-Nachricht direkt in der Befehlszeile zu schreiben.

  • git status

    • Der Befehl zeigt den aktuellen Zustand des Arbeitsverzeichnisses und der Staging-Area in einem Git-Repository an. Es ist ein nützlicher Befehl, um den Überblick über die Änderungen zu behalten, die Sie seit dem letzten Commit vorgenommen haben.

  • git diff

    • Der Befehl wird verwendet, um die Unterschiede (Diffs) zwischen verschiedenen Zuständen von Dateien in einem Git-Repository anzuzeigen. Es ist ein nützliches Werkzeug, um den genauen Inhalt von Änderungen zu überprüfen, bevor diese in einem Commit gespeichert werden.

  • git restore

    • Der Befehl wird verwendet, um die Unterschiede (Diffs) zwischen verschiedenen Zuständen von Dateien in einem Git-Repository anzuzeigen. Es ist ein nützliches Werkzeug, um den genauen Inhalt von Änderungen zu überprüfen, bevor diese in einem Commit gespeichert werden.

  • git branch branchname

    • Der Befehl git wird verwendet, um einen neuen Branch in einem Git-Repository zu erstellen. Dieser neue Branch ist eine Kopie des aktuellen Branches, auf dem Sie sich befinden.

  • git branch -a (Kurzform für --all)

    • Der Befehl git branch -a zeigt eine Liste aller Branches in einem Git-Repository an, einschließlich sowohl der lokalen als auch der entfernten Branches. Der Schalter -a ist die Kurzform für --all.

  • git checkout

    • Der Befehl wird verwendet, um zu einem anderen Branch zu wechseln oder eine bestimmte Version von Dateien oder Commits wiederherzustellen.

  • git checkout name

    • Wechselt zu einem bestehenden Branch mit dem Namen name.

  • git checkout -b name

    • Der Befehl wird verwendet, um einen neuen Branch zu erstellen und gleichzeitig zu diesem neuen Branch zu wechseln. Dies ist eine gängige Methode, um in Git einen neuen Branch zu beginnen, der auf dem aktuellen Branch basiert.

  • git switch -c feature-x

    • In neueren Versionen von Git können Sie auch git switch verwenden, um einen neuen Branch zu erstellen und zu wechseln:

    git checkout -b name: Erstellt einen neuen Branch mit dem Namen name und wechselt sofort zu diesem neuen Branch.

  • git push

    • Der Befehl wird verwendet, um Ihre lokalen Commits auf ein entferntes Git-Repository (Remote) zu übertragen. Dies ist der Prozess, bei dem Sie Ihre Änderungen, die Sie lokal gemacht haben, auf den Remote-Server hochladen, damit andere Personen diese Änderungen sehen und mit ihnen arbeiten können.

  • git pull

    • Der Befehl git pull wird verwendet, um Änderungen von einem Remote-Repository in Ihr lokales Repository zu holen und diese Änderungen automatisch in den aktuellen Branch zu integrieren. Er kombiniert zwei wichtige Git-Befehle: git fetch (um die neuesten Änderungen vom Remote-Repository herunterzuladen) und git merge (um diese Änderungen in den aktuellen Branch zu integrieren).

  • git merge

    • Der Befehl wird verwendet, um die Änderungen von einem Branch in einen anderen Branch zu integrieren. Dies ist ein zentraler Bestandteil des Workflows in Git, insbesondere wenn Sie mehrere Branches verwenden, um verschiedene Features, Bugfixes oder Versionen zu entwickeln.

  • git revert

    • Der Befehl wird verwendet, um die Änderungen eines spezifischen Commits rückgängig zu machen, indem ein neuer Commit erstellt wird, der die Änderungen des angegebenen Commits aufhebt. Dieser Befehl ist nützlich, wenn Sie eine fehlerhafte Änderung oder einen Commit rückgängig machen möchten, ohne die Historie des Projekts zu verändern.

  • git revert --abort

    • Der Befehl wird verwendet, um einen Revert-Vorgang abzubrechen, wenn er nicht abgeschlossen werden kann oder wenn während des Revert-Vorgangs Konflikte auftreten, die Sie nicht lösen möchten. Dies ist nützlich, um den aktuellen Revert-Versuch zurückzusetzen und den Zustand des Repositories vor dem Revert-Versuch wiederherzustellen.

  • git log --oneline --graph

    • Der Befehl git log --oneline --graph ist ein nützlicher Git-Befehl, der eine kompakte und visuelle Darstellung der Commit-Historie eines Repositories bietet. Dieser Befehl zeigt die Historie der Commits in einer vereinfachten Form an und verwendet ASCII-Grafiken, um die Branches und Merge-Historie zu visualisieren.
